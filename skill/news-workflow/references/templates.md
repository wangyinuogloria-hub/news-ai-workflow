# 稿件结构模板

## 一、动态新闻模板（1'30" ~ 2'）

### 模板 A：经济/外贸类（最常用）

```
【标题】（模式A/B/C均可）
【导语】定位句 + 核心数据 + 意义提炼（2-3句）
【正文1】场景描写开头 → 企业/现场细节（2-3句）
【同期声1】企业负责人：经营数据/市场判断
【正文2】"据介绍"/"数据显示" → 行业背景或政策支撑（2-3句）
【同期声2】海关/官员：政策解读/监管数据
【正文3】数据总结 → 行业趋势 → 深圳地位（3-4句）
【同期声3】协会/专家（可选）：行业展望/宏观判断
```

**代表稿件：** 储能出海、机器人出口、跨境电商

---

### 模板 B：口岸/消费类

```
【标题】事件概括
【导语】时间 + 事件 + 意义（1-2句）
【正文1】场景描写 → 现场观察（2-3句）
【同期声1】旅客/消费者：亲身体验
【同期声2】官员/负责人：政策背景/数据
【正文2】补充数据 + 趋势分析（2-3句）
【同期声3】企业/商户（可选）
```

**代表稿件：** 免税店、口岸客流、跨境消费

---

### 模板 C：文化活动/事件类（简约型）

```
【标题】事件概括
【导语】时间 + 地点 + 事件 + 意义（1-2句）
【正文1】事件经过 + 亮点细节（2-3句）
【同期声1】参与者/负责人：核心观点
【正文2】（可选）→ 背景补充或意义升华
```

**代表稿件：** 新版《梁祝》、校地共建、电池平台上线

---

### 模板 D：民生/节日类

```
【标题】事件概括（偏温馨/生动）
【导语】节日的意义 + 深圳的做法（1-2句）
【同期声1】记者出镜/现场声（可选）
【正文1】场景A描述 → 活动现场细节
【同期声2】参与者（群众）：亲身体验
【正文2】场景B描述 → 转场 → 另一类活动
【同期声3】参与者（群众）：感受
【同期声4】组织者/负责人：活动意义
【正文3】数据总结 → 社会意义（2-3句）
【同期声5】（可选）专家/策展人
```

**代表稿件：** 六一活动

## 二、主题报道模板（3' ~ 5' 或更⻓）

### 模板 E：多子题并列式（论坛/峰会综合稿）

```
【主标题】核心事件
【导语】时间 + 事件 + 宏观意义（2-3句）
【正文】总体介绍或第一部分正文

---（分隔标记）

【子标题1】角度A
【正文】该角度下的现象/事实
【同期声1-3】多位专家/嘉宾（同话题不同视角）

---（分隔标记）

【子标题2】角度B
【正文】该角度下的现象/事实
【同期声4-6】多位专家/嘉宾

---（分隔标记）

【子标题3】角度C（以此类推）
...
```

**代表稿件：** 文化强国论坛（主论坛+分论坛）

---

### 模板 F：深度产业分析式（进出口总额类主题报道）

```
【标题】总体概括
【导语】核心数据 + 定性判断（2-3句）
【同期声/现场声】记者出镜（建立在场感）或一线人员

---（分隔标记）

【子标题1】细分角度A
【正文】事实 + 数据
【同期声1】企业负责人
【同期声2】海关官员

---（分隔标记）

【子标题2】细分角度B
【正文】事实 + 数据
【同期声3】企业负责人
【同期声4】海关官员

---（分隔标记）

【子标题3】细分角度C
【正文】事实 + 趋势
【同期声5】行业专家/协会负责人（定调收尾）
```

**代表稿件：** 前四月深圳进出口规模继续保持内地城市首位

## 三、结构原则总结

### 3.1 同期声穿插节奏
- 正文段落不宜连续超过 2 段（否则配音过长，观众疲劳）
- 同期声之间的正文段落起"承上启下"作用
- 第一个同期声通常出现在稿件前 1/3 位置

### 3.2 信息密度
- 每段正文 2-4 句（适配 15-25 秒配音）
- 核心数据在导语和正文中各出现一次（首尾呼应）
- 同期声不重复正文已有的信息

### 3.3 段落衔接
- 场景切换用正文（配音）实现，不用同期声直接跳场景
- "另一边""同时""而在..." 是高频转场词
- 子标题之间独立成篇，但保持主题统一

### 3.4 开头和结尾
- **开头**：场景/数据/政策三者必居其一，不用空泛的"近日，..."
- **结尾**：专家/协会同期声收尾 > 趋势展望 > 数据总结（优先级递减）
