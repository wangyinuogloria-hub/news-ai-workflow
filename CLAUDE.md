# 电视新闻 AI 辅助写稿工作流

## 项目说明

本项目的核心是在 Claude Code 中实现一套完整的电视新闻稿件自动写作工作流，覆盖选题 → 背景调查 → 素材录入 → 同期声处理 → 角度策划 → 事实核查 → 写稿修改的全流程。

用户为地方电视台新闻栏目的记者，新闻调性聚焦本地科技、文化、经济等领域的蓬勃发展。

**当前阶段：** 三方案并行 — Claude Code Skill（满血版）/ Streamlit Web 应用（折中版）/ Dify 移动端（入门版）。

## 工作流启动方式（Claude Code 版）

按照七步流程逐步推进，每一步由用户触发：

1. **用户说"开始选题"** → 进入 Step 1（选题录入）
2. **用户提供选题信息后** → 进入 Step 2（背景调查）
3. **用户开始叙述采访经历** → 进入 Step 3（场景素材录入）
4. **用户粘贴同期声文本** → 进入 Step 4（同期声录入 & 素材全景摘要）
5. **Step 4 完成后** → 进入 Step 5（角度策划 & 同期声框架）
6. **用户选定角度后** → 进入 Step 5.5（同期声确认 & 时间戳输出）
7. **确认同期声后** → 进入 Step 6（事实核查清单）
8. **用户确认核查清单后** → 进入 Step 7（自动写稿 & 对话修改）

**注意：**
- 每步需用户主动触发，AI 不自动推进
- 如用户同时提供多个步骤的信息，AI 依次执行
- 记忆库更新贯穿始终，不在特定步骤中

## 核心规则

### 同期声处理铁律
- ✅ 可拼接同一人的多段同期声
- ✅ 可调换同期声先后顺序
- ✅ 可删减口头禅、重复词
- ❌ 不可修改原文措辞
- ❌ 不可将不同人的话拼接成一句
- ❌ 不可断章取义歪曲原意
- ❌ 不可编造不存在的同期声

### 同期声引用格式
`姓名 职位` — 姓名后空格接职位，无括号

### 背景调查溯源
每条数据、政策、案例必须附可访问的原始链接

## 记忆库更新机制

每次用户投喂新材料时，AI 自动：
1. 提取风格特征 → 更新 `memory/style-guide.md`
2. 提取专有名词 → 更新 `memory/terminology.md`
3. 分析同期声选择 → 更新 `memory/quote-patterns.md`
4. 提取结构模板 → 更新 `memory/article-templates.md`

记忆库文件为持久化知识，跨对话保留。

**Dify 同步：** 记忆库更新后，需定期导出文件上传至 Dify 知识库。
